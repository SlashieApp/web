{/* Reads the resolved value of a Chakra color token from the live theme. */}
export function resolveToken(role) {
  if (typeof window === 'undefined') return ''
  const cssVar = `--chakra-colors-${role.replace(/\./g, '-')}`
  return getComputedStyle(document.documentElement)
    .getPropertyValue(cssVar)
    .trim()
}

export function Swatch({ role, note }) {
  const value = resolveToken(role)
  const isDark = /surface|canvas|subtle|raised|onGreen/.test(role)
  return (
    <Stack gap={2} minW="0">
      <Box
        h="56px"
        borderRadius="lg"
        bg={role}
        borderWidth="1px"
        borderColor="border.default"
      />
      <Stack gap={0.5} minW="0">
        <Text fontFamily="mono" fontSize="xs" fontWeight={600} color="text.default">
          {role}
        </Text>
        {note ? (
          <Text fontSize="xs" color="text.muted">
            {note}
          </Text>
        ) : null}
        <Text fontFamily="mono" fontSize="xs" color="text.subtle">
          {value || '…'}
        </Text>
      </Stack>
    </Stack>
  )
}

# Slashie Design Language

The **Slashie Design Language (SDL)** is defined in a single source of truth:
[`src/theme/chakraSystem.ts`](?path=/docs/foundations-overview-principles--docs).
Everything below is read from the theme at render time, so this page can never
drift from the shipped tokens — change a value in code and the docs update with it.

---

## 1. Two-tier tokens

SDL has exactly two layers of color:

1. **Primitives** — raw scale values (`green.400`, `neutral.900`, `status.danger.solid`).
   These are the paint. **Feature code never references them.**
2. **Semantic roles** — the only thing components and pages reference
   (`action.primary`, `bg.surface`, `text.muted`, `border.focus`, …).

Light and dark ship as two systems that share **identical primitives**; only the
semantic role *values* differ. That is why a theme toggle "flips semantic tokens,
primitives unchanged". Toggle the **Theme** control in the toolbar above — every
swatch and example on this page re-resolves live.

> **Rule:** in feature code, reference a semantic role only. Never a primitive
> (`green.400`) and never a raw hex. Raw consumers (Mapbox, SVG, meta tags) use
> `src/theme/brand.ts` constants instead.

## 2. Green-ink rule

`action.primary` is green-400 (`#00DC82`) — too light to carry white text and
pass contrast. So **every green fill pairs with `text.onGreen`** (a near-black
ink, `#0A1512`), never white. The same applies to solid danger fills.

<HStack gap={3} my={5} flexWrap="wrap">
  <Button variant="primary">Post a task</Button>
  <Button variant="danger">Cancel task</Button>
  <Button variant="premium">Go premium</Button>
</HStack>

<Text fontSize="sm" color="text.muted">
  Primary and danger fills above use <code>text.onGreen</code> ink. Premium (plum)
  is dark enough for white. White on green or red fails AA and is never used.
</Text>

---

## 3. AA contrast

Every text/background pairing targets **WCAG 2.1 AA** (4.5:1 for body text,
3:1 for large text and UI affordances). The ink-on-green pairing clears AA
comfortably (≈10:1), which is why the green-ink rule exists. The a11y addon
(panel below each story) flags any pairing that regresses.

---

## 4. Status is never color alone

Color is never the *only* signal for state — that excludes color-blind and
low-vision users. Every status chip carries a **dot or icon plus a text label**.
Use the `Badge` `dot` prop. Domain adapters (e.g. task lifecycle chips) compose
`Badge` with a fixed family + label map — they do not live in `@ui`.

## 5. Reduced motion

Transitions use `sdlMotion` and animate **transform / opacity only** (never
layout). Always honour `prefers-reduced-motion: reduce` and drop non-essential
movement. The `emphasis` easing is reserved for confirmation moments (quote
accepted, job closed).

## 6. Map = list

Anything shown on a **map** must also be available as an accessible **list**.
Maps are not keyboard- or screen-reader-friendly on their own, so the list is
the source of truth for interaction and the map is a spatial enhancement on top.
The dedicated `duration.map` token (below) keeps map transitions consistent.
