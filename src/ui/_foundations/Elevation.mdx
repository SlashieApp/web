# Elevation

SDL ships a single elevation scale, `sdlElevation` (`e1`–`e5`), exported from
`@/theme/styles` and registered as Chakra `shadows` tokens in
`src/theme/chakraSystem.ts`. Reference elevation by token (`boxShadow="e3"`),
never as a raw `box-shadow` string in feature code. The legacy
`xs/sm/md/lg/xl` shadow names are kept as aliases of `e1`–`e5` during migration.

Every value below is read live from the theme modules, so this page cannot
drift from code.

export const LEVELS = Object.keys(sdlElevation)

## Dark mode: prefer border over heavy shadow

`sdlElevation` shadows are tuned for the light canvas (a low-alpha ink shadow on
white). On the dark canvas a soft shadow against a dark surface reads as muddy,
so dark mode communicates depth primarily through a **raised surface color plus
a 1px border**, not a heavy drop shadow:

- Step the surface up the `bg` ramp (`bg.canvas` → `bg.surface` → `bg.raised`)
  rather than increasing shadow weight.
- Keep a `1px` `border.default` (or `border.strong` for the top layer) so edges
  stay crisp without relying on the shadow.
- It is fine to keep the same `boxShadow` token in both modes — it simply does
  less visual work in dark, where the border and surface carry the elevation.

Notice in the comparison above how each card stays legible in dark via its
border and surface, while in light the shadow does most of the lifting.

## Motion

Animate elevation changes with `sdlMotion` (from `@/theme/styles`) and only ever
animate `transform`/`opacity`/`box-shadow` — never layout properties. The cards
above transition over `{sdlMotion.duration.moderate}` using the
`standard` easing curve (`{sdlMotion.easing.standard}`).
